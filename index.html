一、目标

在自己电脑上部署一个AI对话大模型，并实现GUI。（该大模型使用的是Deepseek r1）

二、基本技术路线

  1. 使用Ollama来引入大模型。

  2. 使用Open WebUI来创建GUI界面。

三、安装Ollama

直接到Ollama官方网站安装即可，无需绿色上网。

Ollama官网：Ollama官网 下载页面：Ollama下载

Ollama会默认安装在C盘。安装完毕后，可以使用环境变量调整Ollama的安装位置和默认的模型位置和本身的位置。

安装成功后可以在托盘看到一个小羊驼的图标，也可以在CMD中输入

ollama

来查看是否安装成功。

1. 调整安装位置

（1） 将C:\Users\XX\AppData\Local\Programs\Ollama这个文件夹移动到其他盘，例如改为D:\Ollama。

（2）将修改用户变量的PATH变量，将原来的C:\Users\XX\AppData\Local\Programs\Ollama路径更新为新的位置，即D:\Ollama。

2. 调整模型储存位置

编辑或创建一个新的用户账户变量 OLLAMA_MODELS，设置为你希望存储模型的路径。

踩坑：将Ollama文件位置移动后，Ollama的程序位置就改变了，这个时候再从搜索里搜索Ollama来打开托盘图标会没有任何反应。这个时候要手动修改一下快捷方式的指向位置。另外，快捷方式应指向ollama app.exe而不是ollama.exe。

四、安装Open WebUI

Open WebUI可以给我们提供一个比命令行更美观的GUI界面。如果你觉得命令行交互也不是不行，可以选择不安装。

Open WebUI只能通过Docker进行安装。所以我们在安装Open WebUI之前要先安装Docker。

1. 安装Docker

Docker是运行在Linux系统上的，所以在安装Docker之前，我们还要完成Windows的Linux虚拟化。

这里有两种解决方案：

  1. 使用基于Hyper-V的虚拟化。

  2. 使用基于WSL的虚拟化。

我安装是使用方案2，基于WSL的虚拟化。所以先讲一下方案2。

（1） 虚拟Linux环境

方案一：基于WSL的虚拟

打开命令行，输入：

wsl --install

即可一键安装WSL2。

注意：此时需要科学上网。

提示：本人安装时，即使科学上网，输入命令后也报错：无法解析服务器。原因是raw.githubusercontent.com 这个域名DNS解析不到IP地址。解决方式：

  1. 修改DNS解析。改位默认114.114.114.114，备用8.8.8.8。（实测无效）

  2. 修改host文件。

    （1）打开IP或域名解析网站（例如IP在线解析网站 IP138），解析raw.githubusercontent.com这个域名的IP地址（本人解析结果为185.199.110.133）。
    然后ping一下，确保能ping通。

    ping 185.199.110.133

    （2）打开host文件（host文件在C:\Windows\System32\drivers\etc这个目录下），将解析结果复制进去。

    185.199.110.133 raw.githubusercontent.com

    （3）再次ping一下raw.githubusercontent.com能ping通就说明没问题了

    ping raw.githubusercontent.com
方案二：基于Hyper-V的虚拟

在控制面板中找到程序和功能，选择启用或关闭Windows功能，找到Hyper-V，选择即可。

操作完成后，可以在任务管理器查看虚拟化是否已经启动。

（2）安装Docker_Desktop

官网：Docker_Desktop官网，下载地址：Docker_Desktop下载地址，直接下载即可。

注：主流的桌面PC，笔记本电脑，服务器（包括虚拟机）都在用X86_64的CPU，也就是选择AMD64安装包。

安装时直接选择跳过选择啊注册啊什么有的没的就行。由于我是安装完毕后才写的笔记，所以没有截图，直接skip就行。

2.部署Open WebUI

在科学上网的环境下，使用命令行一键部署。打开CMD，输入以下命令：

注意：此时一定要科学上网，否则速度感人。

docker run -d -e HF_ENDPOINT=https://hf-mirror.com -p 3000:8080 --add-host=host.docker.internal:host-gateway -v [安装目录]:/app/backend/data --name open-webui --restart always ghcr.io/open-webui/open-webui:main

[Ollama安装目录]要替换为自己的安装目录。比如这里安装到D盘。

docker run -d -e HF_ENDPOINT=https://hf-mirror.com -p 3000:8080 --add-host=host.docker.internal:host-gateway -v D:\APP\ollama-web-ui:/app/backend/data --name open-webui --restart always ghcr.io/open-webui/open-webui:main

等待下载完毕，就全部部署成功了。在Docker_Desktop里可以看到open-webui的服务。

点击Port(s)即可打开对话界面。
（首次打开会有一个注册界面，注册一下就可以了。）
